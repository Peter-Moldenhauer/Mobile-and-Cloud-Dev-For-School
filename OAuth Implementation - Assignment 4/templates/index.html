OAuth 2.0 Implementation


This application demonstrates OAuth 2.0 Server Side flow. The following pieces of functionality are implemented in this application:

  • A page that has a link a user clicks on to visit the Google OAuth 2.0 endpoint
  • A page that handles the user getting redirected back to the website from Google's endpoint and handles the exchange of the access code for a token
  • A page (which can use the same handler as the page the user was redirected to) which uses that token to access and display the users first and last name and the URL to access the users Google+ account. It will also display the value of the state variable that was used to secure the original redirect.

To get started, click on the button below. Once the button is clicked on, this app will request access to protected resources (email) on your Google+ account. As noted above, once access to the protected resource has been gained, the following information will be displayed: first and last name, URL to access your Google+ account, and the value of the state variable that was used to secure the original redirect.
